## 검증 목적

1차 테스트는 Docker 기반 테스트 환경을 만들고, 우공실 서비스가 컨테이너 환경에서 어떤 방식으로 실행되는지 확인하기 위한 기준 테스트입니다. 로컬 실행과 배포 환경 사이의 차이를 줄이고, 이후 반복 테스트에서 비교할 수 있는 기본 구조를 만드는 데 초점을 둡니다.

## 중점 정리 항목

- 테스트 목적과 사용 도구 정리
- 테스트 전용 폴더 구조 구성
- Docker와 Docker Compose 기반 실행 흐름 확인
- 백엔드, 프론트엔드, DB 연결이 부하 테스트 환경에서 함께 동작하는지 확인
- 초기 부하 상황에서 확인해야 할 서버 응답, CPU, 메모리, 오류 로그 기준 정리

## 저장소 공개 방식

이 저장소에는 테스트 원문 전체를 복제하지 않고, 배포 전 성능 검증 근거로 활용할 수 있는 요약만 남깁니다. 상세 화면, 실행 과정, 스크린샷은 원문 링크에서 확인하고, 저장소에는 민감정보가 포함되지 않은 재검증 기준만 유지합니다.

## 후속 확인 항목

- 테스트 조건과 사용 명령어가 재현 가능한 형태로 정리되어 있는지 확인합니다.
- 부하 발생 시 서버가 비정상 종료되지 않는지 확인합니다.
- DB 연결 수, API 응답 지연, 정적 파일 제공 병목을 다음 테스트에서 비교할 수 있도록 기록합니다.
