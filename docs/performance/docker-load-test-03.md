## 검증 목적

3차 테스트는 앞선 테스트에서 확인한 조건을 바탕으로 배포 전 안정성을 다시 확인하는 기록입니다. 저장소 공개와 AWS Lightsail 반영을 앞두고, Docker 환경에서 확인한 성능 이슈가 실제 운영 배포 판단에 어떻게 연결되는지 정리하는 데 초점을 둡니다.

## 중점 정리 항목

- 1차와 2차 테스트에서 확인한 병목 후보 재점검
- 반복 요청 상황에서 백엔드 서버가 안정적으로 유지되는지 확인
- 프론트엔드 빌드 결과와 백엔드 API가 함께 운영 가능한지 확인
- 부하 상황에서 관리자 기능, 문제 풀이, 게시판, 회식맵 같은 주요 기능에 영향이 없는지 확인
- Lightsail 반영 전 추가로 확인해야 할 운영 체크리스트 정리

## 저장소 공개 방식

3차 테스트는 배포 전 성능 검증의 최종 참고 자료로 연결합니다. 저장소에는 배포 판단에 필요한 요약과 체크리스트를 유지하고, 원문 블로그는 테스트 과정의 상세 근거로 둡니다.

## 후속 확인 항목

- Lightsail 환경의 Node.js, npm, MySQL 버전과 로컬 검증 환경의 차이를 확인합니다.
- 운영 서버에서 `.env` 값, DB 연결 제한, 프로세스 실행 방식이 테스트 조건과 맞는지 확인합니다.
- 배포 후에는 서버 로그, DB 연결 수, 응답 지연, 메모리 사용량을 일정 시간 모니터링합니다.
