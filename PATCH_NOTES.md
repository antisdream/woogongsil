# 우공실 변경 이력

이 문서는 사용자가 체감하는 기능 변화와 운영 안정성 개선을 날짜별로 정리한 기록입니다. 내부 서버 키, 운영 DB 덤프, 개인 백업 경로, 비밀 환경 변수는 포함하지 않습니다.

## 2026-06-20

### FIX20 보안, 접속 보호, 관리자 운영 안정화

#### 접속 보안

- 첫 접속 기기에서는 게이트키퍼 인증을 거치도록 유지했습니다.
- 게이트키퍼 인증을 통과한 기기에서는 로그인, 회원가입, 계정 찾기 등에서 hCaptcha 반복 입력을 줄였습니다.
- 캐시 삭제, 기기 변경, 브라우저 변경, 인증 쿠키 만료 시에는 다시 보안 확인이 필요할 수 있습니다.
- 모바일 로컬 접속 환경에서 게이트키퍼 인증 쿠키가 저장되지 않던 문제를 보정했습니다.

#### API 보호

- 사용자의 주요 데이터 변경 API가 클라이언트에서 보낸 `userId`만 믿지 않도록 개선했습니다.
- 오답노트, 학습 결과, 운세 기록, 게시판, 댓글, 맛집, 멀티플레이, 관리자 API는 서버 세션과 권한을 다시 확인합니다.
- 다른 사용자의 ID를 개발자도구에서 임의로 넣어 조회하거나 수정하려는 요청은 차단됩니다.
- 주요 쓰기 API와 조회 증가 API에 요청 제한을 추가했습니다.

#### 자동화와 크롤러 방어

- 보안 헤더를 추가해 iframe 삽입, MIME sniffing, 과도한 referrer 노출을 줄였습니다.
- Content Security Policy를 적용해 허용된 스크립트, 이미지, 지도, hCaptcha 리소스 범위를 명확히 했습니다.
- `robots.txt`와 `X-Robots-Tag`를 적용해 검색 엔진과 자동 크롤러의 색인화를 제한했습니다.
- CORS 허용 범위를 제한해 브라우저 기반 외부 origin 요청을 줄였습니다.

#### 의존성 보안

- backend와 frontend의 npm audit 취약점을 0개로 정리했습니다.
- `nodemailer`, `axios`, `react-router-dom`, `vite`, `postcss` 등 직접 의존성을 안전 버전으로 갱신했습니다.
- `ws`, `qs`, `form-data`, `follow-redirects`, `js-yaml`, `@babel/core` 등 하위 의존성은 취약 범위 밖 버전으로 고정했습니다.

#### 성능과 번들

- 프론트엔드 라우트를 지연 로딩해 초기 메인 번들 크기를 줄였습니다.
- 기존 1000kB 초과 번들 경고가 사라졌고, 주요 페이지는 필요한 시점에 나뉘어 로드됩니다.

#### 관리자 화면

- 회원 목록의 최근 로그인/최근 로그아웃 의미를 확인했습니다.
- 최근 로그인은 로그인 성공 시각이고, 최근 로그아웃은 명시적 로그아웃 API 호출 시각입니다.
- 브라우저 종료, 서버 재시작, 탭 종료는 명시적 로그아웃으로 기록되지 않을 수 있습니다.
- `로그인 유지`와 `현재 접속중`은 다른 개념이므로, 이후 화면에서 더 명확히 표시할 개선 후보로 정리했습니다.

#### 운영 검증

- 운영 서버 반영 전후로 전체 서버 파일과 DB 백업을 생성했습니다.
- 앱 DB와 MySQL 전체 DB 덤프를 모두 보관했습니다.
- SHA256 검증을 통해 AWS 백업과 PC 다운로드 백업이 일치함을 확인했습니다.
- 운영 반영 후 `/`, `/admin`, `/mealmap`, `/robots.txt`, `/api/ipep/health`가 정상 응답하는 것을 확인했습니다.

## 2026-06-19

### FIX19 화면 테마, 맛집 지도, 문제 랜덤 개선

#### 화면과 테마

- 다크 모드와 라이트 모드에서 배경, 글자, 카드, 버튼 색상이 자연스럽게 보이도록 정리했습니다.
- 화면 밝기를 10% 단위로 조절할 수 있도록 개선했습니다.
- 밝기 조절이 일부 영역에만 어색하게 적용되는 문제를 줄이고, 전체 화면 톤이 함께 맞춰지도록 보정했습니다.
- 홈 화면의 메인 문구와 주요 안내 문구 위치가 안정적으로 보이도록 정리했습니다.
- 로그인, 회원가입, 마이페이지, FAQ, 게시판, 운세, 문제 풀이 화면의 안내 문구와 버튼명이 더 읽기 쉽도록 정리했습니다.

#### 문제 풀이

- 필기와 실기 랜덤 문제에서 방금 풀었던 문제가 바로 다시 나올 가능성을 줄였습니다.
- 전체 과목 랜덤 풀이에서는 최근에 나온 과목이 과도하게 반복되지 않도록 개선했습니다.
- 문제 수가 부족한 경우에는 조건을 완화해 문제 풀이가 멈추지 않도록 처리했습니다.
- 실기 문제, 기출 문제, 오답노트의 안내 문구와 결과 문구를 더 명확하게 정리했습니다.

#### 맛집 지도

- 사용자가 장소를 제보하면 관리자 승인 대기 없이 지도에 공개되도록 변경했습니다.
- 장소 수정이나 삭제 요청은 기존처럼 관리자 검토를 거쳐 반영되도록 유지했습니다.
- 장소 제보 화면의 깨진 문구와 어색한 안내 문구를 정리했습니다.
- 지도 열기 버튼명을 실제 사용 흐름에 맞게 정리했습니다.
- 카테고리 입력과 추천 목록을 개선했습니다.
- 지도, 검색창, 상세 패널, 댓글 입력창이 라이트/다크 모드와 밝기 조절에 맞게 보이도록 보정했습니다.

#### 회원가입 승인

- 신규 회원가입은 바로 가입 완료되지 않고 관리자 승인 대기 상태로 접수되도록 변경했습니다.
- 관리자가 승인하면 실제 계정이 생성되고, 승인 안내 메일이 발송됩니다.
- 관리자가 거절하면 거절 사유가 포함된 안내 메일이 발송됩니다.
